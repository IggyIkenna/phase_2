Executive Summary

Why we’re here

The Elysium contract includes a Phase Two “Production Implementation” deliverable. The issue is that “production” can mean very different things: a minimal system to execute one strategy vs a robust platform with execution safeguards, routing, simulation, and operational tooling.

This memo sets a clear definition for each path so delivery, costs, and responsibilities are aligned.

Decision
  • Option 1 — $125k: complete Phase 2 with a tightly bounded “production” definition (single basis strategy; minimal execution stack)
  • Option 2 — $195k: build the full execution + infra platform (MEV/simulation + routing + backtest→live + multi-strategy expansion)

Additional work delivered beyond Annex A

Multi-venue funding research (not “pick one exchange”)

  • We collected and compared funding from Binance, Bybit, OKX (not a single venue assumption)
  • Example finding: Bybit averaged ~8% while OKX was closer to ~0–2% — this materially changes expected carry and allocation
  • We implemented a weighted multi-venue funding series to reduce single-venue bias and operational concentration risk
  • Allocation used in the delivered research package: Binance 25% + Bybit 50% + OKX 25%
  • Data integrity stance: NO fallbacks (analysis fails if data is missing)

Source: delivered research package (multi-venue funding + allocation methodology).

Strategy breadth (more than a single “basis trade”)

  • Delivered analysis covered 16 strategy variants across 6 strategy families
  • Benchmarked against Ethena sUSDe using real data
  • Found that simple staking alone is weak; performance comes from leverage + reward stacking and careful execution assumptions

Source: delivered research package (strategy ROE analysis).

Why live trading expands scope

Execution
  • Atomicity vs partial fills
  • MEV + slippage defense
  • Multi-venue coordination
  • Latency + reliability
Risk
  • Delta drift, liquidation risk
  • Position/PnL reconciliation
  • Guardrails + kill-switches
  • Audit trails
Ops
  • Deployment + secrets
  • Monitoring + alerting
  • Incident response
  • Runbooks + handover

Typically, the difference between “research code” and “production trading” is 2–5× effort for the same strategy.
